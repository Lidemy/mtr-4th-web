實驗 X 自學 X 公開

程式導師實驗計畫開始於 2017 年 12 月，大概每半年為一期， 當初創立的目標為：「四個月內，把你培養成一個找得到工作的網頁工程師（前端或後端）」， 一直到第二期結束後發現時間太短，因此改為六個月。

  • 這是一堂會不斷進化的課程

    計劃名稱會有「實驗」兩個字，是因為課程結束以後會誠實地做檢討（並且對外公開檢討成果），把不足的部分補齊。實驗兩字不代表這堂課不完整，它代表著「實驗精神」，不斷去試錯，不斷去改進，讓課程變得更好，學生也會一起變得更好。

  • 基礎紮實與自學能力才是生存之道

    在這個技術演進飛快的年代，掌握好這兩點才是生存之道。這裡一樣教你如何使用工具，但也會一併把背後原理教給你。課程規劃在中後期刻意不給太詳細的資訊，就是為了讓學生培養自學能力。

  • 正視自己的缺點並公開出來

    「公開」是這個計畫不同於其他人之處。 每一期的就業率、學生人數、薪水資訊以及檢討報告都是公開資訊—— 因為你所看到的缺點，在下一期都會有所改進。課綱以及作業也都公開在 GitHub 上，若是對課程內容有疑慮，可以請周遭熟悉技術的朋友幫你看看課綱品質如何。

歷來改進心得

每一期計畫結束以後，都會花一點時間檢討，並且在下一期改進。 以第三期為例，在課程結束後與十幾位同學開了一次期末聚會兼檢討會，一週一週檢討課程， 從學生的回饋中尋求改進方法。秉持著公開透明的原則，每一期的缺點以及可以改進的事項都會公開在網路上。

  • 2018/9

    第二期改進內容

    • 調整課綱，讓學習曲線不要那麼陡峭
    • 強化程式基礎
    • 新增淘汰制，增加學習動力
  • 2019/4

    第三期改進內容

    • 時間由四個月增長為六個月
    • 新增 JavaScript 核心課程內容，強化 JavaScript 能力
    • 新增部分線上課程，從原本看直播存檔學習調整為看影片
    • 新增自行開發學習系統，增強課程凝聚力
    • 新增網路基礎課程，幫助理解網路相關知識
  • 2020/5

    第四期改進內容

    • 重新調整作業，增加連貫性
    • 以漸進增強的作業取代零散小作業
    • 作業專案完整度提高
    • 引進助教制度，讓助教一起幫忙改作業
    • 修正 PHP 後端課程內容
    • 引入產品以及開發流程相關課程
